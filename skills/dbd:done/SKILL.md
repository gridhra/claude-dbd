# dbd:done

## 動作

1. 今日のタスクファイルを読み込み
2. 未完了タスクの一覧を表示
3. ユーザーに完了タスクと成果を確認
4. ファイルを更新（`[x]` + `_done:{TODAY}` 付与、Doneセクションに追記）

## 対話フロー

### ステップ1: 現状表示

未完了タスクを番号付きで表示:
```
現在の未完了タスク:
1. [ ] API実装 #backend
2. [ ] ドキュメント作成 #docs
3. [ ] PRレビュー #review
```

### ステップ2: 完了確認

```
完了したタスクは？ (番号をカンマ区切りで入力、なければ 'none')
> 1, 3

やったことのサマリーは？（簡潔に）
> ユーザー認証APIを実装、チームメイトのPRをレビュー・マージ
```

### ステップ3: ファイル更新

変換前:
```markdown
- [ ] API実装 #backend
```

変換後:
```markdown
- [x] API実装 #backend _done:2026-03-25
```

Doneセクションに追記:
```markdown
## Done
- ユーザー認証APIを実装
- チームメイトのPRをレビュー・マージ
```

## 手順

1. 今日の日付を取得し、タスクファイルを読み込み:
   ```bash
   date +%Y-%m-%d
   ```

2. 未完了タスク（`- [ ]` の行）を抽出して番号付きで表示

3. AskUserQuestionで以下を確認:
   - 完了したタスクの番号
   - 成果のサマリー

4. ファイルを更新:
   - 該当タスクの `- [ ]` を `- [x]` に変更
   - `_done:{TODAY}` を行末に追加
   - Doneセクションに成果を追記

5. 更新後のファイル内容を表示

## 出力形式

- 更新後のタスクリスト（完了状態含む）
- Doneセクションの新規エントリ
- カウント: 「今日 X 件のタスクを完了！」
