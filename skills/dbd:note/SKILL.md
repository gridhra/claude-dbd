# dbd:note

引数: $ARGUMENTS

## 動作

1. 今日のタスクファイルを確認（なければ `/dbd:today` を案内）
2. **引数なし**: 対話モードでHandoff/Notesを選択
3. **引数あり**: 内容を分析してHandoff/Notesに自動振り分け
4. タイムスタンプ付きで記録

## 対話フロー（引数なしの場合）

AskUserQuestionで確認:
- 「何を記録しますか？」
  - 「Handoff/所感」 → 終業時の振り返り、明日への引き継ぎ、優先事項
  - 「メモ」 → ブロッカー、アイデア、メモ

**Handoff選択時**: 3つの質問を順に（Enterでスキップ可）:
1. 「今日の所感は？」
2. 「引き継ぎ事項は？」
3. 「明日の優先事項は？」

**メモ選択時**: 自由テキスト入力

## 自動振り分けキーワード（引数ありの場合）

以下のキーワードがあればHandoffセクションへ:
- 明日, 次回, 引き継ぎ, 優先, tomorrow, next, handoff, priority

それ以外はNotesセクションへ

## セクション形式

### Handoffセクション

```markdown
### HH:MM
#### 所感
- 内容

#### 引き継ぎ
- 内容

#### 明日の優先
- 内容
```

### Notesセクション

```markdown
- HH:MM 内容
```

## 手順

1. 日付と時刻を取得:
   ```bash
   date +%Y-%m-%d
   date +%H:%M
   ```

2. 今日のタスクファイルを読み込み

3. ファイルがない場合:
   - 「今日のタスクファイルがありません。`/dbd:today` を先に実行してください。」と案内
   - 終了

4. 引数がない場合、AskUserQuestionで種類を確認:
   - 「Handoff/所感」か「メモ」かを選択

5. **Handoff選択時**:
   - 3つの質問を順に実行（空回答でスキップ可）
   - `## Handoff` セクションに構造化された形式で追記

6. **メモ選択時**:
   - メモ内容を確認
   - `## Notes` セクションに `- HH:MM <内容>` で追記

7. **引数あり**:
   - キーワード判定（明日, 次回, 引き継ぎ, 優先, tomorrow, next, handoff, priority）
   - キーワードあり → Handoffセクション（引き継ぎ/優先事項として）
   - キーワードなし → Notesセクション

8. ファイルを保存

## 出力形式

**Handoffの場合:**
```
HH:MM にHandoffを記録:
- 所感: [内容]
- 引き継ぎ: [内容]
- 優先: [内容]
```

**メモの場合:**
```
HH:MM にメモを記録:
- [内容]
```
