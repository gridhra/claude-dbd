# dbd:report

引数: $ARGUMENTS

## 動作

1. 対象日のタスクファイルを読み込み（デフォルト: 今日）
2. 活動ログを分析し、主要な成果を抽出
3. ユーザーに確認・補足を対話で依頼
4. 残タスクから明日の優先事項のみフィルタ
5. Slack mrkdwn形式で日報を生成
6. クリップボードにコピー
7. Slack MCPで投稿（チャンネルIDはプロジェクトCLAUDE.mdの `slack_daily_report_channel:` から取得）

## 日付判定（27時制）

03:00より前は前日扱い:
```bash
HOUR=$(date +%H)
if [ "$HOUR" -lt 3 ]; then
  TODAY=$(date -v-1d +%Y-%m-%d)
else
  TODAY=$(date +%Y-%m-%d)
fi
```

## 手順

1. 対象日を決定（引数があればその日付、なければ27時制で判定）

2. タスクファイルを読み込み

3. ファイルがない場合:
   - 「タスクファイルが見つかりません: {DATE}」と案内して終了

4. **GitHub活動を自動収集**:
   - `/dbd:gh-done` スキルを呼び出して対象日のPR活動をDoneセクションに反映
   - 既にDoneに記録済みのPRは重複スキップされる

5. ファイルを再読み込みして解析・抽出:
   - Doneセクションのエントリ（活動ログ）
   - 未完了タスク（期限付き）
   - Notesセクション

6. **ユーザーへの確認**（通常のテキスト出力で質問し、返答を待つ）:

   ```
   ## 日報作成: {DATE}

   ### 現在の記録

   **Done:**
   - [Doneセクションの内容を表示]
   - （空の場合は「記録なし」）

   **Handoff:**
   - [Handoffセクションの内容を表示]
   - （空の場合は「記録なし」）

   ---

   ### 追加・修正があれば教えてください

   - 他にやったこと
   - 明日への引き継ぎ
   - 成果物の添付（スクショやコードなど）

   **なければ「ok」と入力してください。**
   ```

   返答に応じて:
   - 「ok」「特になし」等 → そのまま日報生成
   - 追加内容あり → Done/Handoffセクションを更新してから生成

7. 成果サマリーを生成（1-2文）

8. 残タスクのフィルタ（明日以前が期限のもののみ）

9. 日報を生成

10. **投稿前確認（AskUserQuestion）**:
   - 生成した日報の全文をCLI上にプレビュー表示
   - その直後にAskUserQuestionで確認:
     - 質問: 「この内容でSlackに投稿しますか？」
     - 選択肢: 「投稿する」「修正してから投稿」「投稿せずコピーだけ」
   - 「修正してから投稿」の場合 → ユーザーの指示に従い修正し、再度プレビュー+確認
   - 「投稿せずコピーだけ」の場合 → クリップボードコピーのみで終了

11. クリップボードにコピー:
    ```bash
    echo "{REPORT_CONTENT}" | pbcopy
    ```

12. Slackに投稿:
    - プロジェクトCLAUDE.mdから `slack_daily_report_channel:` の値を取得
    - 値がなければ投稿をスキップし、「`slack_daily_report_channel` がCLAUDE.mdに未設定です」と案内
    - Slack MCPの `slack_send_message` で投稿
    - 投稿後、メッセージリンクを表示

## 出力形式（Slack mrkdwn）

Slackは標準Markdownではなく独自の「mrkdwn」形式を使用する。

### テンプレート

以下のテンプレートを厳密に守ること。各セクション間は空行1行で区切る。

```
*日報 MM/DD (曜日)*

*本日の成果*
1-2文の成果サマリー

*活動ログ*
• 活動1
• 活動2
• 活動3

*明日やること*
• タスク1 (MM/DD)
• タスク2 (MM/DD)
```

### Slack mrkdwn 記法ルール

| 要素 | 正しい書き方 | 間違い |
|------|-------------|--------|
| 太字 | `*text*` | `**text**` |
| 箇条書き | `• ` (中黒ドット+半角スペース) | `- ` |
| 見出し | `*太字*` のみ | `# ` `## ` や絵文字付き見出し |

### 重要な注意

- **絵文字は使わない**。見出しは `*太字*` のみで十分
- 箇条書きは必ず `• ` （中黒ドット+半角スペース）で開始
- 各セクション見出し（`*太字*`）の前後に空行を入れる
- メモが空ならそのセクションは省略
- 曜日は日本語（月/火/水/木/金/土/日）
