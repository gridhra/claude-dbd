# dbd:report

引数: $ARGUMENTS

## 動作

1. 対象日のタスクファイルを読み込み（デフォルト: 今日）
2. 活動ログを分析し、主要な成果を抽出
3. ユーザーに確認・補足を対話で依頼
4. 残タスクから明日の優先事項のみフィルタ
5. Slack mrkdwn形式で日報を生成

## 手順

1. 対象日を決定:
   - 引数があればその日付を使用
   - なければ今日: `date +%Y-%m-%d`

2. タスクファイルを読み込み

3. ファイルがない場合:
   - 「タスクファイルが見つかりません: {DATE}」と案内して終了

4. ファイルを解析して抽出:
   - Doneセクションのエントリ（活動ログ）
   - 未完了タスク（期限付き）
   - Notesセクション

5. **ユーザーへの確認**（通常のテキスト出力で質問し、返答を待つ）:

   ```
   ## 日報作成: {DATE}

   ### 現在の記録

   **Done:**
   - [Doneセクションの内容を表示]
   - （空の場合は「記録なし」）

   **Handoff:**
   - [Handoffセクションの内容を表示]
   - （空の場合は「記録なし」）

   ---

   ### 追加・修正があれば教えてください

   - 他にやったこと
   - 明日への引き継ぎ
   - 成果物の添付（スクショやコードなど）

   **なければ「ok」と入力してください。**
   ```

   返答に応じて:
   - 「ok」「特になし」等 → そのまま日報生成
   - 追加内容あり → Done/Handoffセクションを更新してから生成

6. 成果サマリーを生成:
   - 活動ログを分析し、1-2文に要約
   - 日報の冒頭に配置

7. 残タスクのフィルタ:
   - 明日以前が期限のタスク（超過含む）のみ表示
   - それ以降のタスクは含めない

8. 日報を生成して表示

## 出力形式（Slack mrkdwn）

Slackは標準Markdownではなく独自の「mrkdwn」形式を使用する。

```
:memo: *日報 MM/DD (Weekday)*

:white_check_mark: *本日の成果*
[1-2文の成果サマリー]

:pencil: *活動ログ*
• 活動1
• 活動2

:arrow_right: *明日やること*
• タスク1 (MM/DD)
• タスク2 (MM/DD)

:speech_balloon: *メモ*
[メモがあれば記載、なければセクションごと省略]
```

### Slack mrkdwn 記法ルール

| 要素 | 正しい書き方 | 間違い |
|------|-------------|--------|
| 太字 | `*text*` | `**text**` |
| 斜体 | `_text_` | `*text*` |
| 箇条書き | `• ` または `* ` | `- ` |
| 見出し | `*太字*` + 絵文字 | `# ` `## ` |

### 絵文字ガイド

- `:memo:` - 日報タイトル
- `:white_check_mark:` - 成果
- `:pencil:` - 活動ログ
- `:arrow_right:` - 明日やること
- `:speech_balloon:` - メモ
- `:warning:` - 注意事項・ブロッカー

### ポイント

- *太字+絵文字*でセクション見出しを作る（`#`は使えない）
- 箇条書きは `•` を使用（`-` より視認性が良い）
- 日付は `MM/DD` 形式（短く）
- メモが空ならセクションごと省略

## クリップボードコピー

日報生成後、自動的にクリップボードにコピー:

```bash
echo "{REPORT_CONTENT}" | pbcopy
```

コピー後に通知: 「クリップボードにコピーしました」
