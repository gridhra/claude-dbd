# dbd:add

引数: $ARGUMENTS

## 動作

1. 今日のタスクファイル（`tasks/YYYY/MM/YYYY-MM-DD.md`）を取得
2. ファイルがなければ `/dbd:today` を先に実行するよう案内
3. **引数あり**: 入力を分析し、複数タスクがあれば分割
4. **引数なし**: AskUserQuestionでタスク内容を対話的に確認
5. `due:DATE` がないタスクはユーザーに期限を確認
6. Tasksセクションに追記
7. 追加内容を確認表示

## 手順

1. ファイル存在確認:
   ```bash
   test -f tasks/$(date +%Y)/$(date +%m)/$(date +%Y-%m-%d).md
   ```

2. ファイルがなければ「`/dbd:today` を先に実行してください」と案内

3. **引数がある場合**:
   - 入力テキストを分析
   - 複数タスクの判定（箇条書き、番号付きリスト、カンマ、改行、「と」「and」等）
   - タスクごとにタグ（`#tag`, `due:DATE`）を保持して分割

4. **引数がない場合**: AskUserQuestionでタスク内容を質問

5. **期限確認**（`due:DATE` がないタスク）:
   - AskUserQuestionで期限を確認
   - 選択肢: 「今日」「明日」「今週中 (金曜)」「来週中」「期日なし」
   - 複数タスクに期限がない場合は一括で確認（無関係なタスクなら個別に）
   - `due:YYYY-MM-DD` タグを付与

6. ファイルを読み込み、Tasksセクションに追記:
   ```markdown
   - [ ] {タスク1}
   - [ ] {タスク2}
   ```

7. ファイルを保存

## 出力形式

- 追加したタスクの一覧
- 今日のタスク総数
- 確認メッセージ（例: 「3件のタスクを追加」）
