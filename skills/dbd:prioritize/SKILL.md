# dbd:prioritize

## 動作

1. 今日のタスクファイルから全未完了タスクを収集
2. 各タスクの営業日残を計算（土日・祝日を除外）
3. effort/impactタグを分析（なければタスク内容から推定）
4. 優先度スコアで並び替え
5. 不明なeffort/impactがあればユーザーに確認
6. 今日の推奨アクション（上位3件）を提示

## 優先度スコア計算

- **緊急度 (0-40点)**: 営業日残に基づく
- **インパクト (0-40点)**: C=40, H=30, M=20, L=10
- **effortボーナス (0-3点)**: 小さいタスクほどクイックウィンとして加点

## 手順

1. 今日の日付を取得してタスクファイルを読み込み

2. 各未完了タスクについて:
   - `due:DATE` から暦日・営業日残を計算
   - `effort:X` と `impact:X` タグを確認
   - タグがなければタスク内容から推定
   - 優先度スコアを算出

3. スコア降順で並び替え

4. 優先度テーブルを表示:
   ```
   | 優先度 | タスク | 期限 | 営業日残 | effort | impact |
   |:------:|--------|------|:--------:|:------:|:------:|
   | 🔴 | タスク名... | 03-30 | 2日 | M* | H* |
   ```
   （* はAI推定値を示す）

5. effort/impactが不明で推定もできないタスクがある場合:
   - AskUserQuestionで確認
   - effort: S/M/L/XL、impact: L/M/H/C
   - 回答後、タスクファイルにタグを反映

6. 今日の推奨アクション（上位3件）を表示

## 出力形式

```markdown
# タスク優先度分析 (YYYY-MM-DD)

## 優先度順タスク一覧

| 優先度 | タスク | 期限 | 営業日残 | effort | impact |
|:------:|--------|------|:--------:|:------:|:------:|
| 🔴 | タスクA | 03-30 | 2日 | M | H |
| 🟠 | タスクB | 03-31 | 3日 | S* | M* |
...

## 情報が不足しているタスク

- タスク名...
  → 不明: effort, impact

## 今日の推奨アクション

1. **タスクA** ⚠️
   - 営業日残: 2日
2. **タスクB**
   - 営業日残: 2日
3. **タスクC**
   - 営業日残: 3日

---
effort: S(数時間) / M(1日) / L(数日) / XL(1週間+)
impact: L(低) / M(中) / H(高) / C(クリティカル)
```

## 祝日設定

祝日は `.local/holidays/YYYY.json` から読み込み:

```json
{
  "year": 2026,
  "country": "JP",
  "holidays": [
    { "date": "2026-01-01", "name": "元日" }
  ]
}
```

このファイルはgitにコミットしない（`.gitignore` に含まれる）。
